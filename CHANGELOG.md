# 更新日志

本项目遵循面向开源自部署用户的发布记录。维护者发布新版本时，请把用户需要知道的功能变化、修复、数据库迁移和环境变量变化写在这里。

## 未发布

- 积分明细页：侧边栏新增「积分明细」入口（`/credits`），展示可用/冻结积分、累计入账/出账合计，以及全部收支流水（充值、赠送、消费、冻结、返还，含说明、时间、变动额与结余）。冻结（下单前临时占用）仅作占用、随后由消费或返还结算，故不计入收支合计以免重复计数；数据按账号隔离、仅显示本人记录。复用既有 `CreditTransaction` 数据表，无需数据库迁移。

- 提示词标签检索：「提示词库」页新增标签筛选栏，按引用热度列出热门标签（引用次数降序），点击一个或多个标签即按「同时命中所有选中标签」（AND）逐步收窄结果，可与分类、关键词搜索叠加；卡片上的标签也可直接点击加入筛选，一键「清除」重置。复用既有 `PromptTag` 数据表，无需数据库迁移。

- VIP 会员中心：顶栏「VIP 升级会员」按钮此前为占位提示，现指向全新会员页（`/membership`）——展示当前会员状态（套餐名、有效期、剩余天数）、三项核心权益（生成折扣 / 每日赠送积分 / 更高生成频率，随后台会员配置实时显示）、以及可开通/续费的会员套餐（仅列出订阅型套餐 `SUBSCRIPTION`）。选择套餐与支付渠道后复用既有下单接口（`POST /api/billing/orders`）创建订单并跳转支付；无可用在线支付渠道时给出提示并引导至充值中心。会员开通/续期沿用既有支付回调逻辑（订单支付成功即续期 `Subscription`），本次仅新增前台入口页，无需数据库迁移。

- 风格预设库：创作页新增「风格预设」一栏，内置赛博朋克 / 极简主义 / 复古胶片 / 水彩插画 / 3D 渲染 / 日系动漫等风格，点击即把该风格的提示词修饰（正向后缀 + 可选负向后缀）追加到当前描述，再次点击可撤销，帮助新手快速出片。预设由运营维护（数据表 `StylePreset`，随 `prisma db seed` 内置默认若干条），前台仅展示启用中的预设并按排序显示。新增数据表 `StylePreset`（需执行数据库迁移，并建议运行 `prisma db seed` 初始化默认风格）。

- 图片一键二创（生成变体）：「生成历史」每张输出图新增「生成变体」按钮，点击后系统将该图复刻为你的参考图，并携带原任务的提示词/负向词/比例/质量/张数跳转到创作页，一步进入「以图生图」二次创作。仅可复刻本人历史图或仍公开可见的广场作品（图片不存在/已删除返回 404、无权访问返回 403），源图地址由服务端按图片记录取出、用户仅传图片 ID，因此不存在任意地址抓取（SSRF）风险；复用既有参考图上传全链路（体积/真实类型校验、公共存储、10 分钟 20 次限流），需管理员已开启参考图功能。无需数据库迁移。

- 提示词热门榜：「提示词库」页顶部新增「热门提示词榜」，按收藏与浏览热度排出前若干名（收藏数优先，其次浏览量），带名次徽章、分类、收藏/浏览计数与一键「去创作」，帮助快速发现大家都在用的模板（全 0 的冷门模板不占榜）。同时点击模板卡片的「去创作」会以埋点方式记录一次使用（`navigator.sendBeacon`，失败不影响跳转），让榜单真实反映使用热度。复用既有 `Prompt` 表的 `viewCount`/`favoriteCount` 字段，无需数据库迁移。

- 生成历史导出：「生成历史」页新增「导出 CSV / 导出 JSON」按钮，一键下载你的生图任务清单（任务ID、状态、通道、中/英文提示词、负向词、比例、质量、张数、积分、错误信息、创建时间、图片地址）。CSV 带 UTF-8 BOM，Excel 直接打开不乱码，逗号/引号/换行字段按 RFC 4180 转义；数据仅含当前登录用户本人记录。无需数据库迁移。

- 提示词模板库：侧边栏新增「提示词库」入口（`/prompts`，此前为占位跳转），上线可浏览的提示词模板库——按分类筛选、关键词搜索（标题/简介/标签），每张卡片可一键「去创作」（自动带入中/英文提示词与负向词到创作页）或复制提示词。登录用户可收藏喜欢的模板并用「只看收藏」快速回看，收藏数实时维护。复用既有 `Prompt`/`PromptCategory`/`PromptTag`/`PromptFavorite` 数据表，无需数据库迁移。

- 站内消息通知：顶栏新增通知铃铛（登录可见），未读数以红点角标展示、每分钟轮询刷新；点开可查看通知列表、单条点击即跳转并标记已读，或一键「全部已读」。当他人点赞/评论你发布到画廊的作品时（仅限你本人生成的公开作品，运营精选无作者不触发；自赞/自评不通知），会收到一条通知，跳转到「我的生成记录」。通知写入为「尽力而为」，任何异常都不影响点赞/评论主流程。新增数据表 `Notification`（需执行数据库迁移）。

- 个人资料与头像：新增「个人资料」页（账户菜单 → 个人资料，`/profile`），登录用户可设置昵称与上传头像。昵称最多 24 字、留空则回退用邮箱前缀显示；头像支持 PNG/JPG/WEBP、不超过 4MB，以文件真实内容识别类型（拒伪造 Content-Type），复用现有公共存储桶。设置后头像即时显示在顶栏账户菜单与画廊评论中（此前评论头像位始终为占位首字母）。复用 `User` 表既有 `displayName`/`avatarUrl` 字段，无需数据库迁移。

- 画廊点赞与评论：画廊广场（含运营精选）作品卡片右下角展示点赞数与评论数，登录用户在作品详情弹窗内可一键点赞/取消点赞，并发表、查看、删除（仅限本人）评论。评论内容复用现有内容安全审核（关键词/语义），命中即拦截并落审计日志；点赞与评论均要求作品当前仍公开可见（下架/删除后不可再互动）。点赞幂等（一人一赞），计数实时更新。新增数据表 `GalleryImageLike`、`GalleryImageComment`（需执行数据库迁移）。

- 每日签到领积分：顶栏新增「每日签到」入口——登录用户点击后弹出签到弹窗，展示连续签到进度、今日可得积分、累计签到天数与累计获得积分，一键签到即时到账。连续签到奖励更高：首日发放基础积分，之后每多连续一天递增，加成天数封顶后不再增长；断签自动从第 1 天重新累计，同日重复签到幂等不重复发放。是否开启、基础积分、每日递增积分、连续加成封顶天数均可在控制台「安全与存储 → 每日签到」配置（默认关闭，基础 5 / 递增 1 / 封顶 6 天）。奖励在事务内原子发放，`unique(userId, checkInDate)` 保证每日仅一次。新增数据表 `DailyCheckIn`（需执行数据库迁移）。

- 核心钱/安全逻辑的数据库集成测试：在纯逻辑单测之外，新增连真实 Postgres 的集成测试（`npm run test:db`），覆盖用 mock 测不到的「涉及钱」条件更新 SQL 与幂等保证——积分冻结/结算/退款/充值（`reserveCreditsForJob`/`spendReservedCreditsForJob`/`refundReservedCreditsForJob`/`grantPurchasedCredits`，含余额不足 402、重复结算/退款不双扣、reserve→spend/refund 全链路账实一致）、邀请码解析与返积分发放（`resolveReferrerByCode`/`getOrCreateReferralCode`/`grantReferralRewardsInTx`）、内容审核关键词拦截与审计落库、以及语义审核分支（`checkModerationText`：mock DeepSeek 请求，覆盖语义命中拦截落库、放行、模型不可用/网络异常 fail-open、未配置 Key 时不发起请求）。集成测试独立成组：仅当设置环境变量 `DATABASE_URL_TEST` 时运行，未设置则自动跳过（`npm test` 纯逻辑测试保持零数据库依赖）。首次使用需 `createdb image2_app_test` 建独立测试库并执行 `DATABASE_URL=$DATABASE_URL_TEST npm run test:db:setup`。此为纯开发侧改动，不影响运行时行为与部署。

- 引入 Vitest 与核心纯逻辑单测：新增 Vitest 测试基建（`vitest.config.ts`、`tests/setup.ts`，通过 `npm test` 运行），并为一批高价值纯逻辑函数补充单元测试——出图积分估价（`estimateGenerationCreditCost`）、接口限流（`checkRateLimit`）、内容审核关键词匹配与语义判定解析（`parseForbiddenWords`/`checkForbiddenWords`/`parseSemanticVerdict`）、设置项归一化器（布尔/非负整数/会员折扣/保留天数/超时秒数）、邀请码生成（`createReferralCode`），以及参考图 magic-byte 类型探测（`detectReferenceImageMimeType`，验证伪造 Content-Type 无效）。此为纯开发侧改动，不影响运行时行为与部署。

- 邀请返积分：顶栏「邀请有礼」现可用——登录用户点击后弹出邀请弹窗，展示专属邀请码、邀请链接（一键复制）、已邀请人数与累计返积分。新用户通过邀请链接（`/signup?ref=邀请码`）注册成功后，邀请人与被邀请人各得一份积分奖励（叠加在新用户注册赠送积分之上）。是否开启、双方奖励积分数均可在控制台「安全与存储 → 邀请返积分」配置（默认关闭，邀请人 30 / 被邀请人 20）。邀请码惰性生成、去除易混淆字符；奖励在注册事务内原子发放，仅注册时发放一次。`User` 表新增 `referralCode`/`referredById` 字段（需执行数据库迁移）。

- 作品收藏：画廊广场（及运营精选）作品卡片右上角新增心形收藏按钮，登录后可一键收藏/取消收藏；侧边栏新增「我的收藏」页面，集中查看收藏的公开作品与精选，取消收藏后即时移除。收藏仅在作品仍公开可见时生效，原作品下架/删除后自动从收藏列表剔除。新增数据表 `ImageFavorite`（需执行数据库迁移）。

- 登录二步验证（2FA，邮箱验证码）：用户可在用户中心「账号安全 → 二步验证」自助开启/关闭（开启/关闭均需校验当前密码，且需已配置 SMTP 发信）。开启后，无论从前台登录页还是控制台登录，输入正确密码后都会向邮箱发送 6 位验证码，输入验证码方可完成登录（前台登录页与控制台登录页均会自动进入验证码步骤）。验证码 10 分钟内有效、单次消费、错误 5 次作废，发送带 60 秒节流。管理员账号同样适用。`User` 表新增 `twoFactorEnabled` 字段（需执行数据库迁移）；验证码复用 `EmailVerificationCode` 表（purpose=`login_2fa`）。

- 会员到期邮件提醒：会员到期前会自动发送提醒邮件，到期后再发送一次到期通知（复用邮件模板体系，含续费入口）。提醒提前天数可在控制台「安全与存储 → 会员权益 → 到期提醒提前天数」配置（默认 7 天，0 关闭，需已配置 SMTP 发信）。去重按会员到期时间记录，续费后自动为新周期重新提醒；扫描随账单概览接口惰性触发（进程内每小时至多一次），也可另配 cron 调用 `sendDueMembershipReminders`。`Subscription` 表新增 `expiringRemindedFor`/`expiredRemindedFor` 字段（需执行数据库迁移）。

- 邮件模板体系：系统邮件统一改用带品牌页头/正文卡片/页脚的响应式模板（`src/lib/email-templates.ts`），页头站点名与 logo 取自「系统设置」的站点标题/Logo，注册验证码与 SMTP 测试邮件已迁移到新模板，观感更一致、更接近正式产品邮件。同时内置会员到期提醒邮件模板，供后续到期提醒功能复用。

- 内容审核审计日志：内容安全拦截（关键词/语义）现会写入审计日志，控制台「管理后台 → 审核日志」可查看被拦截的时间、用户、方式（关键词/语义）、命中类别/命中词、命中项与提示词，并支持按邮箱/类别/命中词/提示词模糊搜索。仅记录被拦截的请求，放行内容不落库；写日志失败不影响审核与出图主流程。新增数据表 `ModerationLog`（需执行数据库迁移）。

- 第三方账号绑定/解绑：用户中心「账号安全」页新增「第三方账号绑定」区块，已登录用户可把 GitHub / Google 账号绑定到当前账号或解绑（复用现有 OAuth 回调地址，无需新增回调登记）。绑定时校验第三方账号未被他人占用、同一渠道不重复绑定；解绑时若账号未设置登录密码且这是最后一种登录方式则拒绝，避免锁死账号。管理员账号不支持第三方绑定。

- 会员权益落地：有效会员可享三项权益，均在控制台「安全与存储 → 会员权益」配置——① 出图积分折扣（按 % off 减免，最低仍收 1 积分）；② 每日赠送积分（每天首次出图时自动发放一次，幂等不重复）；③ 出图频率上限提升（普通用户固定每 10 分钟 10 次，会员按配置值）。折扣在下单估价时生效，冻结/结算/退款全链路一致；充值页展示「会员专享权益」清单。`Subscription` 表新增 `lastDailyGrantAt` 字段（需执行数据库迁移）。

- 会员卡（月卡/年卡）：积分套餐新增「套餐类型」，可设为「会员卡（周期）」并配置有效天数（月卡填 30、年卡填 365）。用户购买会员卡后即时发放套餐积分，同时按天数续期会员有效期（未到期时叠加顺延）；充值页展示当前会员状态与到期提醒（剩余 7 天内或已到期高亮提示），到期后手动购买续费。管理沿用现有在线支付渠道，无自动代扣。新增数据表 `Subscription`，`CreditPackage`/`RechargeOrder` 新增 `packageType`/`durationDays` 字段（需执行数据库迁移）。

- 第三方登录（OAuth）：登录/注册页新增「使用 GitHub / Google 登录」入口（控制台「安全与存储 → 第三方登录」配置，默认关闭）。已验证邮箱会自动关联到同邮箱的既有账号，否则创建新用户并发放新用户积分；管理员账号不支持第三方登录。回调地址为 `{站点地址}/api/auth/oauth/{github|google}/callback`，站点地址取 `NEXT_PUBLIC_SITE_URL`。新增数据表 `OAuthAccount`（需执行数据库迁移）。相关环境变量：`OAUTH_GITHUB_CLIENT_ID`、`OAUTH_GITHUB_CLIENT_SECRET`、`OAUTH_GOOGLE_CLIENT_ID`、`OAUTH_GOOGLE_CLIENT_SECRET`。

- 创作页：生成任务状态改为 SSE 实时推送（`GET /api/generation/jobs/{id}/events`），到账更快、请求更省；浏览器不支持或连接异常时自动回退到原有 2s 轮询。离开页面或重新提交会自动断开连接。

- 内容安全：违禁词拦截之外新增「语义审核」（控制台「安全与存储 → 内容安全」开关，默认关闭）。开启后关键词放行的提示词会再调用 DeepSeek 兼容通道做语义判断（复用「生图通道」的 DeepSeek 配置，可单独指定审核模型）。模型不可用/超时时自动放行，不阻断正常生成。相关环境变量：`MODERATION_TIMEOUT_MS`（可选，默认 15000）。

- 存储：新增对象存储支持（S3 兼容协议，一套配置覆盖 AWS S3 / 阿里云 OSS / 腾讯云 COS）。控制台「安全与存储」页签新增 Endpoint / Bucket / Region / AccessKey / 路径风格寻址配置；生成图与参考图上传后落对象存储并回写可访问 URL，支付凭证等私有文件仍走鉴权回源路由（不公开直链）。切回 local 本地存储行为不变。相关环境变量：`STORAGE_PROVIDER`、`STORAGE_ENDPOINT`、`STORAGE_BUCKET`、`STORAGE_REGION`、`STORAGE_FORCE_PATH_STYLE`、`STORAGE_ACCESS_KEY_ID`、`STORAGE_SECRET_ACCESS_KEY`。

- 正式开放参考图生图：控制台「系统设置 → 生图通道」新增「开放参考图生图」开关（默认关闭，也可用环境变量 `REFERENCE_IMAGES_ENABLED` 开启）。开放后创作页支持上传最多 4 张参考图，「再次生成」会完整回填参考图。
- 参考图视觉分析改走所配置的 OpenAI 兼容通道（原先硬编码官方域名，中转通道 Key 会 401 导致分析静默失效），分析模型新增「参考图视觉分析模型」配置项（默认 gpt-4o，留空关闭分析）。
- Stability AI 图生图的引擎段改用后台配置的模型名（原先硬编码 stable-diffusion-xl-1024-v1-0）。
- ChatGPT 网页版通道不支持参考图，带参考图提交时会明确报错提示切换通道，不再静默忽略。
- 安全加固：参考图上传改为服务端读取文件真实内容（magic bytes）判定类型，不再信任客户端声明的 Content-Type，伪造类型的非图片文件会被拒绝。
- 参考图新增过期清理机制：超过保留天数且未被任何生成任务引用的参考图会被软删除并删除磁盘文件；上传成功后惰性触发（进程内节流）。新增「参考图保留天数」配置项（控制台「生图通道」，默认 30 天，也可用环境变量 `REFERENCE_IMAGE_RETENTION_DAYS` 设置）。

- 前台新增暗色模式：顶栏主题按钮支持「浅色 / 暗色 / 跟随系统」三态切换，本地记忆并防止刷新闪烁，桌面与移动端均可用。
- 生成任务轮询在离开创作页或重新提交时自动停止，不再在后台持续请求。
- 控制台系统设置页首次加载显示骨架屏，取代空白等待。

- 性能：后台配置读取增加内存缓存（30 秒），大幅减少每次页面请求对配置表的重复查询；控制台设置页与全站首屏加载更快。修改配置后即时生效。
- 性能：提示词润色不再为读取一个配置项而执行完整的后台诊断（含数据库探测）。
- 性能：后台用户列表的统计计数改为批量聚合查询，用户较多时打开更快（原先每个用户单独查 4 次）。
- 性能：生成图与参考图的回源支持长效缓存与 304 协商缓存，重复浏览走浏览器/磁盘缓存，减轻服务器图片流量。

- 安全：修复"未设置密码的账号可被任意密码接管"的问题——此类账号改为拒绝密码登录，并收紧管理员初始化逻辑，不再静默提权已注册的普通用户或覆盖其密码。
- 安全：API 错误响应不再把数据库/内部异常原文透给前端，仅透传业务提示，其余返回通用文案并记服务端日志。
- 安全：支付凭证文件（storage 的 payments 目录）改为需要本人或管理员会话才能访问，未授权返回 404；系统自检新增"凭证是否暴露在 public 目录"的检查项。
- 支付：修复订单超时过期后收到成功支付却"钱付了积分不到账"的问题——过期订单迟到支付现在可正常入账；已取消订单收到支付会拦截并记诊断供人工处理。
- 支付：取消订单改为原子操作，消除与支付回调竞态时把已支付订单误标为已取消的可能；取消已支付订单会明确提示无法取消。
- 积分：修复冻结积分在重复结算/退款时可能被减为负数或双倍返还的问题（条件更新 + 任务失败状态守卫）。

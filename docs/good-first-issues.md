# Good First Issue 候选清单

这份清单用于维护者持续创建适合新贡献者的 issue。建议每次保持 3 到 6 个 `good first issue`，让首次参与者能快速找到入口。

## 文档类

### 补充一份英文快速部署摘要

目标：在 `README_EN.md` 中补充更完整的 Docker Compose 部署说明，帮助非中文用户理解项目。

验收：

- 不包含真实密钥或生产凭证。
- 与 `README.md` 和 `.env.example` 中的变量名保持一致。
- 包含源码部署和 Docker Compose 的差异说明。

### 整理常见 Docker 部署错误

目标：在 `docs/faq.md` 中补充用户常见 Docker 错误和排查步骤。

可覆盖：

- 端口被占用。
- 数据库连接失败。
- Prisma 迁移权限不足。
- 图片 volume 挂载错误。

### 补充 Provider 配置说明

目标：新增或完善 OpenAI 官方接口、OpenAI 兼容接口和 DeepSeek 润色配置示例。

验收：

- 示例值必须是占位符。
- 不推荐前端直接调用第三方模型接口。
- `chatgpt_web` 必须保持实验性和默认禁用描述。

## UI 和移动端类

### 优化某个移动端表格溢出问题

目标：挑选一个后台或前台页面，修复手机端横向溢出或操作列遮挡。

验收：

- 不影响桌面端布局。
- 不破坏 TDesign 工作台风格。
- PR 里附上修复前后截图。

### 优化弹窗在移动端的安全区域

目标：修复弹窗、Alert 或下拉控件遮挡底部导航的问题。

验收：

- 小屏宽度下不横向溢出。
- 底部导航不遮挡主要操作。
- 保持现有浅色、蓝白、拟 App 风格。

## 开源运营类

### 补充 1 分钟演示脚本

目标：在 `docs/open-source-checklist.md` 或新文档中补充 B 站 / 视频号演示脚本。

脚本建议覆盖：

- 项目定位。
- Docker 快速部署。
- 提示词润色。
- 生图工作台。
- 管理后台配置。

### 整理 Release Notes 示例

目标：基于 `CHANGELOG.md` 的最近版本，补充一份真实可用的 Release Notes 示例。

验收：

- 不包含密钥、内部 token 或用户隐私。
- 明确写出部署影响和升级注意事项。
